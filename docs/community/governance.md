# Governance SimpleRAN

## Structure organisationnelle

SimpleRAN n'a pas de leader formel. C'est une **association non-constituée** (Unincorporated Association) auto-régulée.

```
┌─────────────────────────────────────────┐
│    Libre Endowment Fund (LEF)           │
│    (Juridical structure & tax breaks)   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │   SimpleRAN Contributors        │   │
│  │   - Self-regulated individuals   │   │
│  │   - Companies (vendors, telcos)  │   │
│  │   - Institutions (universities)  │   │
│  │   - Communities (open source)    │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  Governance Principles          │   │
│  │  ✓ Transparent decisions        │   │
│  │  ✓ Community input              │   │
│  │  ✓ Vendor neutral               │   │
│  │  ✓ Open standards               │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

---

## Principes fondamentaux

### 1. **"Do whatever you want, tell us how"**
- Aucune obligation d'implémentation
- BUT : Interfaces doivent être publiques
- Liberté maximale + transparence

### 2. **"Should rather than Must"**
- Recommandations, pas d'impositions
- Flexibilité pour innovation
- Standards de référence (3GPP, ETSI)

### 3. **Vendor Neutrality**
- Pas d'avantage pour un fournisseur
- Niveaux de jeu équal
- Certification objective

### 4. **Pas de NDA**
- Zéro secrets
- Connaissance ouverte
- Confiance communautaire

---

## Prise de décision

### Consensus-based
- Propositions soumises à discussion
- Débat ouvert (mailing list / GitHub)
- Approbation par consensus
- Escalade si désaccord

### Transparency
- Toutes les discussions publiques
- Minutes de réunions publiées
- Décisions documentées
- Archive accessible

## Membership

### Joindre SimpleRAN

1. **Envoyer email** : simpleran@fdl-lef.org
2. **Décrire intentions** : Contributions prévues
3. **Participer** : Discussions, PRs, testing
4. **Pas d'approbation formelle** : Auto-régulation

### Niveaux d'engagement

| Level | Commitment | Rights |
|-------|-----------|--------|
| **Lurker** | Suivre | Lire discussions |
| **Contributor** | Occasionnel | Vote sur issues |
| **Maintainer** | Régulier | Merge PRs |
| **Steering** | Actif | Strategic direction |

## Conformité & Standards

### Certification Process

1. **Specification** : Interfaces publiques
2. **Testing** : Interopérabilité vérifiée
3. **Review** : Communauté évalue
4. **Approval** : Consensus atteint
5. **Listing** : Vendeur certifié

### Comité de certification

- Pas de structure formelle
- Experts reconnus par consensus
- Décisions collégiales
- Appels possibles

---

## Financement

### Libre Endowment Fund (LEF)

- **Juridical entity** : Support légal
- **Tax breaks** : Contributions déductibles
- **No proprietorship** : Pas de contrôle
- **Sustainability** : Long-term viability

### Contributions acceptées

- Donations (cash)
- Code (PRs, repos)
- Infrastructure (servers, bandwidth)
- Expertise (reviews, testing)

---

## Conflits de résolution

### Si désaccord

1. **Discussion** : Expliquer positions
2. **Evidence** : Données/tests objectifs
3. **Escalation** : Mailing list communauté
4. **Consensus** : Chercher accord
5. **Vote** : Si nécessaire (rare)

### Révocation

- Très rare et difficile
- Nécessite consensus large
- Motif : Violation principes clés
- Appels acceptés
